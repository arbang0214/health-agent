# 런로그(RunLog) — 러닝머신 기록 캘린더 설계

작성일: 2026-07-14
상태: 사용자 승인 완료

## 개요

헬스장 러닝머신 계기판 사진을 올리면 기록(운동 시간·거리·칼로리)을 자동 인식하고,
사진의 촬영 일시를 읽어 캘린더에 스티커로 표시하는 **개인용** 운동 기록 웹 서비스.

- 사용자: 본인 1명 (회원가입 차단, 본인 계정만 로그인 가능)
- 접속 환경: 폰 브라우저(헬스장에서 즉시 업로드) + PC 브라우저, 데이터 공유
- 비용: 0원 (Vercel 무료 + Supabase 무료 티어, OCR은 브라우저에서 실행)

## 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js (App Router) | Vercel 무료 배포, 폰/PC 반응형 |
| 호스팅 | Vercel 무료 티어 | 서버 관리 불필요 |
| DB / 인증 / 파일 | Supabase 무료 티어 (Postgres + Auth + Storage) | 한 서비스로 해결, RLS 지원 |
| OCR | Tesseract.js (브라우저 실행) | 무료, 서버 비용 없음 |
| EXIF 파싱 | exifr (브라우저 실행) | 촬영 일시 추출 |

## 아키텍처

```
[브라우저 (폰/PC) — Next.js]
  ├─ 사진 선택 → exifr로 EXIF 촬영 일시 추출
  ├─ Tesseract.js OCR → 시간·거리·칼로리 후보값 파싱
  ├─ 확인·수정 폼 (저장 전 필수 단계)
  └─ 저장 → Supabase
        ├─ Storage(비공개 버킷): 원본 사진 (업로드 전 브라우저에서 리사이즈·압축)
        └─ Postgres: workouts 행 삽입
```

- OCR·EXIF 처리는 전부 클라이언트에서 수행. 서버(Supabase)는 저장·조회만 담당.
- 사진 조회는 서명된 URL(signed URL)로만 접근.
- Supabase RLS로 본인(user_id) 데이터만 읽기/쓰기 허용.

## 화면 구성 (3개)

### ① 캘린더 홈 (메인)
- 월 보기 / 주 보기 토글. 폰 기본 = 주 보기, PC 기본 = 월 보기.
- 운동한 날짜 칸에 **기본 스티커 1종** 표시. 같은 날 기록 2건 이상이면 숫자 배지.
- 상단에 이번 달 요약 한 줄 (예: "이번 달 12회 · 58.2km").
- 하단 큰 + 버튼 → 업로드 화면.

### ② 업로드 화면
- 사진 선택(폰에서는 카메라 직접 촬영 가능) → EXIF + OCR 자동 진행 (스피너 표시).
- 인식 결과가 미리 채워진 폼: 촬영 일시 / 운동 시간(분) / 거리(km) / 칼로리.
- 모든 칸은 탭해서 수정 가능. OCR 실패 시 빈 폼으로 직접 입력.
- 저장하면 캘린더로 복귀, 해당 날짜에 스티커 표시.

### ③ 날짜 상세
- 그날 사진 크게 보기 (여러 장이면 스와이프).
- 사진 아래 기록 수치 표시, 수정·삭제 버튼.

## 데이터 모델

```sql
create table workouts (
  id           uuid primary key default gen_random_uuid(),
  user_id      uuid not null references auth.users(id),
  taken_at     timestamptz not null,  -- EXIF 촬영 일시 또는 직접 입력값
  duration_min numeric,               -- 운동 시간(분), null 허용
  distance_km  numeric,               -- 거리(km), null 허용
  calories     integer,               -- 칼로리, null 허용
  photo_path   text not null,         -- Storage 경로
  created_at   timestamptz not null default now()
);
```

- 수치 3개는 null 허용: OCR 실패 + 미입력 시 "사진만 기록"도 가능.
- 레코드 1건 = 사진 1장.

## 예외 처리

| 상황 | 처리 |
|------|------|
| EXIF 없음 (카톡 전송 사진 등) | 날짜·시간 직접 선택, 기본값 = 현재 시각 |
| OCR 실패·오인식 | 확인 폼에서 수정 (필수 단계이므로 별도 흐름 불필요) |
| 업로드 중 네트워크 끊김 | 실패 안내 + 재시도. Storage 업로드 성공 후 DB 삽입, DB 실패 시 업로드 파일 삭제로 반쪽 저장 방지 |
| 사진 용량 (수 MB) | 업로드 전 브라우저에서 리사이즈(긴 변 1600px)·JPEG 압축 |

## 테스트

- OCR 결과 텍스트에서 시간/거리/칼로리를 골라내는 **파싱 로직 단위 테스트** (다양한 계기판 표기 샘플).
- EXIF 추출·폴백 로직 단위 테스트.
- 실제 러닝머신 계기판 사진 여러 장으로 인식률 수동 확인 후 파싱 규칙 보정.

## 범위 제외 (YAGNI)

- 스트리크(연속 달성), 통계 차트, 스티커 종류 선택, 다중 사용자, 알림 — 필요해지면 추후 추가.
