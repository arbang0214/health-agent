# 런로그(RunLog) — 러닝머신 기록 캘린더 설계

작성일: 2026-07-14
상태: 사용자 승인 완료 (2026-07-14 범위 수정: OCR을 2차로 분리)

## 개요

헬스장 러닝머신 계기판 사진을 올리면 그날 운동한 것으로 기록되어 캘린더에 스티커로 표시되는
**개인용** 운동 기록 웹 서비스. 스티커를 누르면 그날 사진을 상세 조회할 수 있다.

사진 속 기록(운동 시간·거리·칼로리)의 OCR 인식은 **2차 확장**으로, "AI 분석하기" 버튼을 눌러
그동안 올린 사진들을 일괄 분석하는 방식으로 추가한다.

- 사용자: 본인 1명 (회원가입 차단, 본인 계정만 로그인 가능)
- 접속 환경: 폰 브라우저(헬스장에서 즉시 업로드) + PC 브라우저, 데이터 공유
- 비용: 0원 (Vercel 무료 + Supabase 무료 티어, OCR은 브라우저에서 실행)

## 단계별 범위

### 1차 (이번 구현 대상)
- 사진 업로드 → EXIF 촬영 일시 추출(없으면 직접 선택) → 그날 운동 기록으로 저장
- 캘린더 월/주 보기, 운동한 날 스티커 표시
- 스티커(날짜) 클릭 → 그날 사진 상세 조회, 삭제
- 로그인 (본인 계정만)

### 2차 (추후 확장, 이번 구현에서 제외)
- 캘린더에 **"AI 분석하기" 버튼** 추가
- 누르면 그동안 올린(아직 분석 안 된) 사진들을 브라우저에서 Tesseract.js OCR로 일괄 분석
- 시간·거리·칼로리를 인식해 각 기록에 저장하고, 결과 확인·수정 UI 제공
- 2차 착수 시 상세 설계 별도 진행 (데이터 모델은 1차에서 미리 대비)

## 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js (App Router) | Vercel 무료 배포, 폰/PC 반응형 |
| 호스팅 | Vercel 무료 티어 | 서버 관리 불필요 |
| DB / 인증 / 파일 | Supabase 무료 티어 (Postgres + Auth + Storage) | 한 서비스로 해결, RLS 지원 |
| EXIF 파싱 | exifr (브라우저 실행) | 촬영 일시 추출 |
| OCR (2차) | Tesseract.js (브라우저 실행) | 무료, 서버 비용 없음 |

## 아키텍처

```
[브라우저 (폰/PC) — Next.js]
  ├─ 사진 선택 → exifr로 EXIF 촬영 일시 추출 (없으면 직접 선택)
  ├─ 업로드 전 브라우저에서 리사이즈·압축
  └─ 저장 → Supabase
        ├─ Storage(비공개 버킷): 원본 사진
        └─ Postgres: workouts 행 삽입

(2차) 캘린더의 "AI 분석하기" → 미분석 사진들을 signed URL로 받아
      브라우저에서 Tesseract.js OCR → 수치 파싱 → 확인 후 workouts 갱신
```

- EXIF 처리(및 2차 OCR)는 전부 클라이언트에서 수행. 서버(Supabase)는 저장·조회만 담당.
- 사진 조회는 서명된 URL(signed URL)로만 접근.
- Supabase RLS로 본인(user_id) 데이터만 읽기/쓰기 허용.

## 화면 구성 (3개)

### ① 캘린더 홈 (메인)
- 월 보기 / 주 보기 토글. 폰 기본 = 주 보기, PC 기본 = 월 보기.
- 운동한 날짜 칸에 **기본 스티커 1종** 표시. 같은 날 기록 2건 이상이면 숫자 배지.
- 기록이 **없는** 날짜를 탭하면 그 날짜가 미리 선택된 업로드 화면으로 이동 (`/upload?date=yyyy-MM-dd`) — 과거 기록 등록 동선.
- 상단에 이번 달 요약 한 줄 (예: "이번 달 12회"). 거리 합계 등 수치 요약은 2차(OCR) 이후.
- 하단 큰 + 버튼 → 업로드 화면.
- (2차) "AI 분석하기" 버튼이 이 화면에 추가될 예정 — 1차에서는 자리만 고려하고 구현하지 않음.

### ② 업로드 화면
- 사진 선택(폰에서는 카메라 직접 촬영 가능) → EXIF 촬영 일시 자동 추출.
- 촬영 일시가 채워진 확인 폼 (직접 수정 가능). EXIF 없을 때 기본값: 캘린더에서 날짜를 골라 들어왔으면 그날 낮 12시, 아니면 현재 시각. EXIF가 있으면 항상 EXIF 우선.
- 저장하면 캘린더로 복귀, 해당 날짜에 스티커 표시.

### ③ 날짜 상세
- 그날 사진 크게 보기 (여러 장이면 스와이프).
- 촬영 시각 표시, 삭제 버튼.
- (2차) OCR 분석된 시간·거리·칼로리가 여기에 표시되고 수정 가능해질 예정.

## 데이터 모델

2차 OCR 확장을 대비해 수치 컬럼을 처음부터 포함한다 (1차에서는 항상 null).

```sql
create table workouts (
  id           uuid primary key default gen_random_uuid(),
  user_id      uuid not null references auth.users(id),
  taken_at     timestamptz not null,  -- EXIF 촬영 일시 또는 직접 입력값
  duration_min numeric,               -- (2차) 운동 시간(분)
  distance_km  numeric,               -- (2차) 거리(km)
  calories     integer,               -- (2차) 칼로리
  analyzed_at  timestamptz,           -- (2차) AI 분석 완료 시각, null이면 미분석
  photo_path   text not null,         -- Storage 경로
  created_at   timestamptz not null default now()
);
```

- 레코드 1건 = 사진 1장.
- `analyzed_at`이 null인 행이 2차의 "AI 분석하기" 대상이 된다.

## 예외 처리

| 상황 | 처리 |
|------|------|
| EXIF 없음 (카톡 전송 사진 등) | 날짜·시간 직접 선택, 기본값 = 현재 시각 |
| 업로드 중 네트워크 끊김 | 실패 안내 + 재시도. Storage 업로드 성공 후 DB 삽입, DB 실패 시 업로드 파일 삭제로 반쪽 저장 방지 |
| 사진 용량 (수 MB) | 업로드 전 브라우저에서 리사이즈(긴 변 1600px)·JPEG 압축 |

## 테스트

- EXIF 추출·폴백(현재 시각 기본값) 로직 단위 테스트.
- 날짜별 기록 집계(캘린더 스티커·배지) 로직 단위 테스트.
- 폰/PC 브라우저에서 업로드 → 스티커 표시 → 상세 조회 흐름 수동 확인.
- (2차) OCR 파싱 로직 테스트는 2차 설계 시 정의.

## 범위 제외 (YAGNI)

- 스트리크(연속 달성), 통계 차트, 스티커 종류 선택, 다중 사용자, 알림 — 필요해지면 추후 추가.
